# `digital-operating-systems/` — Digital Operating System Architecture

**Status:** Approved

## Purpose

This folder holds the constitutional contract every Digital Operating
System's own Domain Architecture document must follow — its
responsibilities, non-responsibilities, and relationships to Users, the
Core Platform, Shared Ecosystem Services, Infrastructure, and every
other Digital Operating System. It is an unnumbered, cross-cutting
top-level folder rather than a numbered domain folder, per
`00-governance/xp-0003-repository-standards.md`'s Folder Structure
Rules, since this contract spans every Digital Operating System — Business
OS, Community OS, Life OS, and any Future Vertical OS — rather than
belonging to any one of them.

## Documents That Belong Here

- The constitutional contract every Digital Operating System's own
  Domain Architecture document must satisfy — the one document listed
  above.
- Any future non-authoritative Blueprint planning artifact that
  precedes and informs a document at this same cross-cutting tier,
  should one become needed, per `00-governance/xp-0003-repository-standards.md`'s
  Blueprint placement rule.

A specific Digital Operating System's own Domain Architecture document
(Business OS, Community OS, Life OS, or a Future Vertical OS) does not
belong here — each is authored in its own dedicated folder
(`03-business-os/`, `04-community-os/`, `05-life-os/`, and the
Future Vertical OS location still to be assigned, per
`../DOCUMENTATION-ROADMAP.md` §2.10) once authorized.
