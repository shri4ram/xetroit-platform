# 14 — Database

**Status:** Foundation Draft

## Purpose

This folder documents the data architecture and schema standards for the
XETROIT Platform's Laravel modular monolith.

## Documents That Belong Here

- Database technology choices and rationale
- Schema design conventions (naming, migrations, indexing standards)
- Data ownership boundaries between modules within the monolith
- Data retention, archival, and privacy-related data handling rules
- Multi-tenancy data isolation strategy

Module-specific domain models belong in that module's folder as
architecture notes; this folder covers platform-wide data conventions and
standards.
